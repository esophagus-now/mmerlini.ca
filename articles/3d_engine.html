Note there is an optical illustion here: the cube is spinning counter-clockwise (as viewed from the top) but you can make yourself see it spinning clockwise. "Bill of Materials"

I find it's helpful to make a short list of the basic components that will make up the whole system. Here goes:

  • We'll use the HTML <canvas> element to draw on the screen. Specifically, on each frame we'll compute the visuals in a large buffer, then use the putImageData(dat, x, y) function in JavaScript to write it into the canvas.
  • We need a way to "time" our animations. The easiest way would be to somehow trigger our main drawing function at even intervals, but apparently that makes user input difficult. I'll figure out the best choice and come back to it.
  • At this point, the only job left to do is fill in the frame buffer. I'm planning to use the S-buffer technique; the original post I read seems to be taken down, but luckily I've used this technique before and I'll re-explain it here.
  • We'll use the standard matrix and perspective divide techniques to copmute the 2d positions of 3d points.
  • I found a simple scan conversion routine in an old book. I'll describe the method in the relevant article.
  • I want to use either Gouraud or Phong shading. I haven't found anything that explains it simply, so I'll try to figure it out.
  • More importantly, I want my code to have the "vertex shader + fragment shader" model used in openGL in case I want to plug in new shaders down the road.

Okay, that puts my thoughts on the page. I'm gonng do a few quick and dirty tests to find out how I'm gonna do this, then I'll get to work on writing the articles.
